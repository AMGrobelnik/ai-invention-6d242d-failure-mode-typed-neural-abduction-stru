## Full Summary

Two high-quality neuro-symbolic reasoning benchmarks are provided in a unified exp_sel_data_out schema for the FMTNA pipeline evaluation.

**Dataset 1: RuleTaker** (tasksource/ruletaker, IJCAI 2020, Clark et al.)
- 165,833 examples from dev+test splits
- Each example: natural language context (facts + rules), a query question, and an entailment/not-entailment label
- input field: 'Context: {context}\nQuestion: {question}'
- output field: 'entailment' or 'not entailment'
- metadata_config: depth level (depth-0 through depth-5) enabling multi-hop evaluation
- metadata_split: 'dev' or 'test'
- Ideal for proof-failure classification and type-based LLM dispatch

**Dataset 2: FOLIO** (tasksource/folio, EMNLP 2024, Han et al.)
- 1,001 expert-written examples with parallel FOL annotations
- Each example: natural language premises, conclusion, True/False/Uncertain label
- input field: 'Premises:\n{premises}\nConclusion: {conclusion}'
- output field: 'True', 'False', or 'Uncertain'
- metadata_premises_fol and metadata_conclusion_fol: parallel first-order logic formulas
- Unique dual NL+FOL representation enables direct neuro-symbolic grounding

**Why these two datasets:**
- RuleTaker provides large-scale synthetic data (165k examples) with explicit rule structure, configurable reasoning depth (0-5 hops), and clear entailment labels — directly matching the artifact plan's primary dataset.
- FOLIO provides expert-crafted examples with parallel FOL annotations, enabling evaluation of atomic predicate extraction precision and recall against ground-truth first-order formulas.
- Together they cover both scale (RuleTaker) and depth of formal grounding (FOLIO) required for the neuro-symbolic pipeline.

**Output files:** full_data_out split into two parts (<100MB each), plus mini (6 examples, 3 per dataset) and preview (6 examples, truncated). Schema validated against exp_sel_data_out.json.
